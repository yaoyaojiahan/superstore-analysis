- **量化建议**：3 大策略 + ROI 模拟——折扣止血（+$46K）、供应商议价（+$28K）、区域差异化（+$18K），合计可提升利润 32%。

- **可视化**：18 张 Plotly 交互图表 + 3 页 CEO 驾驶舱（含 KPI 警报、热力图矩阵、ROI 模拟器、实施路线图）。

## 版本 B：经营分析 / 运营分析方向

### 零售利润优化与经营诊断项目
*2026.07 | 个人项目*

- **业务背景**：针对年销售额 $230 万的零售企业，开展全链路经营诊断。通过 20 维度分析（12 EDA + 8 下钻），精准定位 $156K 利润泄漏的根因。
- **核心发现**：
  - **利润泄漏**：18.7% 订单亏损，$156K 被侵蚀——但根因不在"卖得不好"，而在"折扣策略失控"
  - **下钻定位**：Furniture × 中折扣段 → 95% 亏损率；Tables × East 区域 → -$11K（单组合最差）；所有客群 × 高折扣段 → 100% 亏损
  - **根因分类**：Binders（折扣驱动，政策可控）vs Tables（区域驱动，需结构性调整）
  - **区域分化**：Central 区域高折扣段亏损 $40,793，是 West 区域的 3 倍
- **策略建议**：
  - 折扣止血：40% 上限 + 8 个"永不打折"清单 → +$46K
  - 产品组合优化：Top 5 亏损品供应商议价 + 末位淘汰 → +$28K
  - 区域差异化：Central 专项诊断 + West 追加投入 → +$18K
- **AI 提效**：Prompt → AI → Python → BI 全流程，提效 5-10×

## 版本 C：AI 数据分析方向

### 零售利润优化智能分析系统（AI + Python + 统计分析 + BI）
*2026.07 | 个人项目*

- **项目定位**：构建从「描述性分析 → 诊断性分析 → 规范性建议」的智能分析 pipeline，为 AI 驱动的自动化经营决策系统奠定基础。
- **技术架构**：
  - `src/data_quality.py` — 自动化数据质量审计（6 项检查，IQR 异常检测）
  - `src/eda.py` — 12 维度 EDA 引擎（Pareto 80/20 分析、利润敏感性分析、客户分层）
  - `src/insights.py` — McKinsey 框架商业洞察生成器（Insight → Recommendation 映射）
  - `src/visualizer.py` — 12 种 Plotly 商业图表工厂函数
- **统计方法**：描述性统计、IQR 异常值检测、帕累托分析、利润率敏感性分析
- **AI 扩展规划**：设计了 ML 增强路线图——时序预测（Prophet）、客户分群（RFM + K-Means）、折扣优化（回归模型）、流失预警（分类模型）
- **工程能力**：模块化代码架构、函数封装、配置分离、PEP 8 规范

> **关键词命中**：Python · 统计分析 · 数据挖掘 · 机器学习路线 · 自动化 pipeline · BI 可视化
