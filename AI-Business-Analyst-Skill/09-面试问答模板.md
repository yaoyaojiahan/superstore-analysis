# 09 — 面试问答模板

> **20 道高频面试题的标准回答框架。覆盖：项目深挖、技术八股、行为面试、业务分析。**

---

## 一、项目深挖题（8 题）

### Q1：简单介绍一下这个项目？

**框架**（STAR）：
- S：业务背景（公司规模、数据量）
- T：分析目标（诊断什么问题）
- A：你做了什么（方法 + 维度 + 工具）
- R：核心成果（数字 + 建议）

**模板**：
"这是一个[项目类型]项目。我用[数据量]条数据，完成了[分析类型]分析。核心发现是[数字+结论]，根因在于[根因]。我给出了[数量]条可执行建议，预计可[量化成果]。技术上用[工具]实现分析，Plotly 做可视化，设计了[N]页驾驶舱。"

---

### Q2：你在这个项目中遇到的最大挑战是什么？怎么解决的？

**框架**：
- 挑战是什么（具体困难，不是笼统的"数据脏"）
- 你怎么想的（分析思路）
- 怎么解决的（方案）
- 学到了什么

**模板**：
"最大的挑战是[具体问题]。我的解决思路是：[思路1] + [思路2] + [思路3]。最终通过[方案]解决了问题。这个经历让我学会了[经验]。"

---

### Q3：你如何判断一条业务建议是否"有价值"？

**框架**（3 标准）：
1. **量化影响**：能带来多大财务改善？（要有数字）
2. **可执行性**：是否具体到操作层面？
3. **优先级**：是否回应了最紧迫的业务问题？

---

### Q4：如果数据量从 1 万行增长到 1000 万行，你的方案如何扩展？

**框架**（分层回答）：
1. 存储层 → PostgreSQL/ClickHouse, 索引, 分区
2. 计算层 → Pandas → Polars/Dask/Spark
3. 可视化层 → 预聚合 → 中间表
4. Pipeline → Airflow/Prefect 调度
5. 架构 → 增量更新替代全量重跑

---

### Q5：你提到用 IQR 方法检测异常值，为什么选 IQR 而不是 Z-score？

**要点**：数据非正态分布（右偏），IQR 基于分位数更稳健。

---

### Q6：帕累托分析在这个项目中的结论是什么？

**框架**：
- 结论（多少品类贡献了多少收入）
- 商业含义（资源聚焦、尾部管理、风险管理）

---

### Q7：你如何向非技术背景的业务方解释 [复杂发现]？

**核心**：用类比替代术语。  
**模板**：
"想象[类比场景]。这就好比[简单比喻]。所以我们需要[行动建议]。"

---

### Q8：如果让你重新做这个项目，你会做什么不同？

**框架**：
1. 加入时间维度 → 趋势/季节性
2. 客户级分析 → RFM/CLV
3. 引入预测 → 销量预测/折扣优化
4. 竞品数据 → 市场份额
5. A/B 测试 → 验证策略效果

## 二、技术八股题（6 题）

### Q9：Python 中 `defaultdict` 和普通 `dict` 有什么区别？

`defaultdict` 访问不存在的 key 自动创建默认值，不需要 `if key in dict` 检查。在做 group-by 聚合时非常有用。

### Q10：你如何用纯 Python 实现类似 Pandas 的 `groupby`？

核心是用 `defaultdict` 分组 + 字典聚合。封装了 `sum_by()`, `avg_by()`, `count_by()` 函数。

### Q11：什么是 Pareto 分析？你在项目中如何实现的？

80/20 法则。实现：排序 → 计算占比 → 累计占比 → 找到 80% 临界点 → 双轴图可视化。

### Q12：Plotly 和 Matplotlib 的区别？为什么选 Plotly？

- Matplotlib：静态、偏学术、代码量大
- Plotly：交互式（hover/zoom/pan）、偏商业、代码量少
- 选 Plotly：项目面向商业场景，交互图表方便展示，HTML 可直接嵌入网站

### Q13：`__init__.py` 文件的作用是什么？

1. 标记目录为 Python 包
2. 定义公共接口（`__all__`）
3. 执行包初始化代码

### Q14：解释你的项目架构设计思路。

分层架构：
- `config/` — 配置层（路径、颜色、常量）
- `utils/` — 工具层（纯函数、可复用）
- `src/` — 业务逻辑层（加载、质量、EDA、下钻、可视化）
- `main.py` — 编排层（串联模块）

## 三、行为面试题（4 题）

### Q15：你如何处理模糊的需求？

先通过提问澄清：业务目标 → 受众 → 时间线 → 数据范围 → 阶段性对齐。

### Q16：你如何确保自己的分析结论是正确的？

数据校验 → 交叉验证 → 常识检验 → 敏感性分析 → 同行评审。

### Q17：描述一次你从数据中发现意外洞察的经历。

（用自己项目的真实发现）意外的是[发现]，这改变了我对[认知]的理解。

### Q18：你如何看待 AI 对数据分析师工作的影响？

AI 增强而非替代：
- AI 擅长：自动化重复劳动、降低技术门槛、加速尝试
- 人不可替代：业务理解、因果推理、沟通说服
- 我的方向：AI + Business Analyst

## 四、业务分析题（2 题）

### Q19：如果你发现某月销售额下降 20%，你会怎么分析？

**拆解框架**：
```
销售额 = 订单量 × 客单价
订单量 = 活跃客户数 × 购买频次
客单价 = 商品数量 × 平均单价
```
然后逐维度下钻：区域 → 品类 → 渠道 → 客户 → 外部因素。

### Q20：给你一家新公司的数据，你会从哪几个维度分析经营状况？

**四维框架**：
1. 收入维度：趋势、结构、客单价
2. 利润维度：毛利率、品类/区域利润贡献、亏损产品
3. 效率维度：库存周转、获客成本、运营费用
4. 客户维度：分层(RFM)、复购率、CLV

根据企业阶段选择优先级：**初创看增长，成熟看利润，转型看效率**。

## 面试技巧总结

| 原则 | 说明 |
|------|------|
| 时间控制 | 每题 1-2 分钟 |
| 框架优先 | "我从 X 个方面回答" |
| 数字说话 | 能用数字绝不用形容词 |
| STAR | 行为题必须 Situation→Task→Action→Result |
| 反问准备 | 准备 2-3 个反问面试官的问题 |
