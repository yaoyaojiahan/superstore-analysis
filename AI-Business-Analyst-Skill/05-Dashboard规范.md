# 05 — Power BI / CEO 驾驶舱设计规范

> **仪表板不是图表画廊，是决策工具。**

---

## 一、设计哲学

| 原则 | 说明 |
|------|------|
| **决策导向** | 每页回答一个决策问题，不是展示图表 |
| **结论前置** | KPI 警报 + 核心发现放在最上面 |
| **少即是多** | 每页 4-6 个模块，不是 12 张图堆砌 |
| **行动驱动** | 必须有"接下来做什么"的明确指引 |

---

## 二、3 页标准结构

### Page 1：经营健康检查（Health Check）

**决策问题**：公司现在健康吗？哪里在出血？

```
┌─────────────────────────────────────────────┐
│ 🔴 CEO 警报条                                │
│ "核心发现：$X 利润泄漏，根因是…"               │
├─────────────────────────────────────────────┤
│ 6 个 KPI 卡片（含目标线 + 红黄绿状态指示）      │
│ [营收] [利润] [利润率] [亏损额] [亏损率] [干预潜力]│
├────────────────────┬────────────────────────┤
│ 品类收入分布        │ 🔴 品类×折扣 热力图        │
│ (Treemap)          │ (核心发现前置！)           │
├────────────────────┴────────────────────────┤
│ TOP 3 干预机会卡片（$ 影响 + 行动）             │
│ ① 折扣上限 → +$X   ② 供应商议价 → +$X         │
└─────────────────────────────────────────────┘
```

### Page 2：根因诊断（Root Cause Diagnosis）

**决策问题**：为什么会这样？问题在哪？

```
┌────────────────────┬────────────────────────┐
│ 子品类×区域亏损矩阵  │ 客群×折扣 利润率对比      │
│ (精准定位问题)       │ (谁被过度打折？)          │
├────────────────────┼────────────────────────┤
│ 亏损品类根因诊断面板  │ 折扣弹性曲线              │
│ (Tables为什么亏?)   │ (最优折扣在哪？)          │
├────────────────────┴────────────────────────┤
│ 🚫 永不打折清单 + 折扣影响柱状图                 │
└─────────────────────────────────────────────┘
```

### Page 3：行动方案（Action Plan）

**决策问题**：现在该做什么？值多少钱？

```
┌─────────────────────────────────────────────┐
│ ROI 模拟器                                    │
│ 当前利润 $X → 执行后 $Y (+Z%)                  │
├─────────────────────────────────────────────┤
│ 3 大策略卡片（量化影响 + 措施列表）              │
│ ① 折扣止血  ② 产品组合  ③ 区域差异化           │
├─────────────────────────────────────────────┤
│ 实施路线图 Q1→Q2→Q3→Q4                        │
├─────────────────────────────────────────────┤
│ 附录：数据质量 + 方法论                         │
└─────────────────────────────────────────────┘
```

## 三、KPI 卡片规范

每个 KPI 卡片必须包含：

```html
<div class="kpi-card [good|warn|danger]">
  <label>指标名称</label>
  <value>当前值</value>
  <target>目标值 / 对比值</target>
</div>
```

颜色规则：
- 🟢 green（accent）：健康 / 达标
- 🟡 amber（warning）：需关注
- 🔴 red（danger）：警报 / 亏损

---

## 四、CEO 警报条规范

放在每页最上方，格式：

```
🔴 核心发现：[一句话量化结论] — [根因]。
执行 [N] 项措施可在 [时间] 内挽回 $[金额]。
```

## 五、图表规范

### 标题规范
每张图表的标题必须是**商业问题**，不是技术描述：

| ❌ | ✅ |
|----|-----|
| Sales by Category Treemap | 哪个品类贡献了最多收入？ |
| Profit Margin by Discount Level | 折扣打到什么程度开始亏钱？ |
| Sub-Category × Region Loss | Tables 在哪个区域亏最多？ |

### 颜色规范
```
主色（Primary）   #003366  深蓝 — 标题、主柱
辅色（Secondary） #0066CC  中蓝 — 次要柱
正面（Accent）    #00A86B  正绿 — 利润、达标
负面（Danger）    #DC3545  警示红 — 亏损、异常
警告（Warning）   #FFC107  琥珀 — 需关注
中性（Neutral）   #6C757D  灰 — 轴标签、副文本
```

### 交互要求
- 所有图表支持 hover 显示详细数据
- 散点图支持 zoom/pan
- 帕累托图必须有 80% 参考线

```

│  $XXX,XXX         +$XX,XXX        +$XX,XXX │
│                     (+X%)           (+X%)   │

```
