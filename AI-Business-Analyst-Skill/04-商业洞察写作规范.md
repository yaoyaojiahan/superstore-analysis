# 04 — 商业洞察写作规范

> **使用 McKinsey "Insight → So What → Now What" 框架，把数据变成决策。**

---

## 一、框架结构

每条商业洞察必须包含 4 个部分：

```markdown
──────────────────────────────────────
  [序号] 维度名称
──────────────────────────────────────

  📊 关联图表：[图表名称]

  💡 商业洞察（Insight）：
     数据发现了什么？用 3-5 句话描述核心发现。

  🎯 商业影响（So What）：
     这对业务意味着什么？为什么 CEO 应该关心？

  ✅ 行动建议（Now What）：
     应该做什么？给出 3-4 条具体措施 + 预计影响。
```

---

## 二、写作要求

### 每个部分的质量标准

| 部分 | 长度 | 标准 |
|------|------|------|
| Insight | 3-5 句 | 必须带数字，不做主观判断 |
| So What | 2-3 句 | 必须连接到业务 KPI，说清楚影响路径 |
| Now What | 3-4 条 | 每条必须有预计影响（$ 或 %），可执行 |

### 数字表达规范

| ❌ 差 | ✅ 好 |
|------|------|
| "利润被侵蚀" | "每 $1 利润被 $0.55 亏损侵蚀" |
| "折扣影响了利润" | ">40% 折扣段利润率 -40.7%" |
| "部分交易亏损" | "18.7% 交易亏损，累计 $156,131" |

### 行动建议规范

每条建议必须：
- **具体**：不是什么"优化折扣策略"，而是"Furniture 品类设 25% 折扣上限"
- **量化**：预计挽回 $XX,XXX
- **有时限**：Q1 执行，Q2 验证
- **有负责人**：运营/管理层/数据团队

| # | 维度 | 核心发现方向 |
|---|------|-------------|

## 四、完整示例

```
──────────────────────────────────────────────────────
  [5] 折扣影响分析
──────────────────────────────────────────────────────

  📊 关联图表：折扣力度 vs 利润率 + 品类×折扣热力图

  💡 商业洞察：
     折扣一旦超过 20%，利润开始转负。>40% 折扣的订单
     平均利润率为 -40.7%，>60% 折扣的订单为 -122.6%。
     更关键的是，Furniture 品类在中折扣段(21-40%)
     就有 95.4% 的订单亏损，涉及 324 笔交易。

  🎯 商业影响：
     这表明当前的折扣策略是系统性的价值摧毁，而非
     促销工具。深度折扣没有带动足够的增量销售来弥补
     利润损失。Furniture 品类在常规促销中就亏损，
     说明定价本身可能存在问题。

  ✅ 行动建议：
     1. 所有品类设 40% 折扣硬上限，>40% 需总监审批
        → 预计挽回 $46,000/年
     2. Furniture 品类额外设 25% 折扣上限
        → 立即止损 324 笔亏损交易
     3. 对 8 个子品类列为"永不打折"
        → 纳入系统管控
     4. A/B 测试找到各品类最优折扣甜点
        → Q2 完成，Q3 全面推广
```

## 五、常见错误

| ❌ 错误 | ✅ 正确 |
|---------|--------|
| 只说"What"不给建议 | 每条 Insight 必须配 Now What |
| 建议空洞（"加强管理"） | 建议具体（"40%上限+审批流程"） |
| 缺少数字 | 每条带量化数据 |
| 缺少预计影响 | 标注预期挽回金额 |
| 写得太长 | 每条 100-200 字，表格优于段落 |
