# 06 — AI 工作流规范

> **从 Prompt 到 BI 呈现：AI 原生分析的标准工作流。**

---

## 一、工作流全景

```
┌─────────────────────────────────────────────────────────┐
│  ① Prompt（中文描述分析需求）                             │
│      ↓                                                  │
│  ② AI 分析设计（框架 + 维度 + 代码骨架）                   │
│      ↓                                                  │
│  ③ Python 执行（数据加载 → 聚合 → 检验）                   │
│      ↓                                                  │
│  ④ 人工校验（合理性 + 业务逻辑 + 洞察质量）                 │
│      ↓                                                  │
│  ⑤ BI 呈现（Plotly → HTML 仪表板 → 咨询报告）             │
└─────────────────────────────────────────────────────────┘
```

---

## 二、每阶段详细规范

### 阶段 ①：Prompt 设计

**好的 Prompt 必须包含 4 个要素：**

```
1. 角色："你是拥有 10 年经验的零售分析专家"
2. 数据："数据集包含 Category, Region, Sales, Profit, Discount 等字段"
3. 目标："不仅要描述发生了什么，还要诊断为什么、给出行动方案"
4. 约束："所有图表中文标题、McKinsey 配色、CEO 报告风格"
```

**标准 Prompt 模板：**

```
你是一个拥有10年经验的[行业]商业分析专家。
数据集包含以下字段：[列出关键字段]。
请设计一个分析方案：
1. 描述性分析维度（至少10个）
2. 诊断性下钻维度（至少6个交叉分析）
3. 每个维度的商业问题
4. 关键KPI定义
5. 预期输出物
```

### 阶段 ②：AI 分析设计

**AI 输出**（人工审核后采用）：

```
分析框架设计 → 代码实现思路 → 预期洞察方向
```

**人工校验清单：**
- [ ] 分析维度是否覆盖所有数据字段？
- [ ] 是否有交叉下钻分析？
- [ ] 商业问题是否清晰？
- [ ] 是否有遗漏的关键维度？

### 阶段 ③：Python 执行

**规范**：
- 使用 `multi_group_by()` 做多维聚合
- 使用 `filter_rows()` 做子集分析
- 每个分析函数返回 dict，方便序列化
- 图表使用 Plotly Graph Objects（非 Express，减少依赖）

### 阶段 ④：人工校验

**校验清单**：

| 检查项 | 标准 |
|--------|------|
| 数据合理性 | 汇总值是否与预期一致？ |
| 业务逻辑 | 发现是否符合商业常识？ |
| 洞察深度 | 是否做了交叉下钻？ |
| 建议质量 | 是否量化了影响？ |
| 语言风格 | 是否 CEO 可读？ |

### 阶段 ⑤：BI 呈现

- Plotly 图表 → 独立 HTML 文件
- HTML 仪表板 → iframe 嵌入图表
- README → 咨询报告风格

## 四、AI 工作流 VS 传统工作流

| 环节 | 传统方式 | AI 方式 |
|------|----------|---------|
| 分析框架 | 分析师凭经验设计 | AI 生成框架 + 人工审核 |
| 代码实现 | 手工写所有代码 | AI 生成骨架 + 人调细节 |
| 洞察提炼 | 人工阅读图表找结论 | AI 辅助标注 + 人验证 |
| 报告输出 | 手工撰写 | AI 生成初稿 + 人润色 |

> **AI 不是替代分析师，而是让分析师把时间花在"校验洞察"和"业务判断"上。**

## 五、实际 Prompt 示例

### 示例 1：分析框架设计

```
作为零售分析专家，我有一个包含 Category, Sub-Category, Region, 
Segment, Sales, Quantity, Discount, Profit 的数据集（约1万行）。
请设计一个多维分析框架，要求：
- 不只要"What"，还要"Why"和"Now What"
- 至少包含品类×折扣、子品类×区域的交叉分析
- 输出每个亏损品类的根因诊断
- 给出可量化的行动建议
```

### 示例 2：代码生成

```
用 Python 实现：对零售数据进行 Category × Discount Bin 的交叉分析。
输入：rows（list of dict），每个 dict 有 Category, Discount, Profit, Sales
输出：3×5 矩阵，每格包含 {margin, loss_rate, count}
要求：使用 multi_group_by 或手动聚合，不使用 pandas。
```

### 示例 3：洞察提炼

```
以下是一个零售数据集的 Category × Discount 交叉分析结果：
[粘贴结果]

请提炼 3 条最关键的商业洞察，每条包含：
1. 数据事实（带数字）
2. 商业含义
3. 行动建议（带预计影响）
```
